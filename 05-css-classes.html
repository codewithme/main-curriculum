All right, let's review a familiar slide.

If you remember from earlier, a selector tells you which tags to affect, like h1. To get technical for one moment, those are called “type selectors.” Basically, those selectors match up with the names of actual HTML elements, like the ones you’ve learned, such as h1 and p.

But you can have multiple paragraphs on a page right? What if you want to style one p tag, but not another? Then a general CSS for p won’t work as well.

That’s where CSS classes come in. They allow you to give your html tags specific names, so you can refer to whichever tags you want to. Let’s look at an example.

Okay, let's see, what's the difference? It's the p.byline section right?

So my goal here is, I want the byline to be gray, but I don't want all my paragraph text to be gray, so I created a special CSS class for it called "byline" because it's a name that's very descriptive. So let's see this step-by-step.

styles.css

   <head>
	  <title>Consider the lobster</title>

HTML elements can also have multiple classes! And it doesn't matter what order you write them in.

   font-size: 20px;
}

.bigger-font {
	font-size: 30px;
